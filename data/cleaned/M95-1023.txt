1:194	Appendix B: MUC-6 Test Score s 291 MUC-6 TEST SCORES This appendix contains the summary score reports for each system in the MUC-6 evaluation.
2:194	The reports are ordered by task (Scenario Template, Template Element, Named Entity, and Coreference) and secondarily by site (i n alphabetical order) . More investigation into system performance may be done on the basis of template-by-template scor e reports, which detail the performance of the system on each article in the test set . The template-by-template score reports are not included in this appendix . A brief introduction to reading the traditional MUC score reports for Scenario Template is presented here.
3:194	Further descriptions of the evaluation metrics and the scoring algorithms used for mapping and scoring may be foun d in the proceedings in the paper entitled "MUC-6 Scoring Procedures and Evaluation Metrics" by Nancy Chinchor and Gary Dungca.
4:194	At the end of this introduction, the variations on this score report format for the three new task s will be discussed.
5:194	The first column in the report, SLOT, contains one of the following types of label : A slot name, e.g. content An object name, e .g. template The word "subtotals" Of the three types of rows, the one of primary interest is the one labeled with a slot name . The rows labeled with an object name may be disregarded . The rows containing subtotals may be of interest to those who want to kno w the performance on the various types of objects in the template . The following tallies for the test set are presented in the leftmost columns of the score report : POS Number possible (COR + INC + MIS), i .e. , the number of fills in the answer key plus any optional fills allowed by the key and generated by the syste m ACT Number actual (COR + INC + SPU),i .e. , the number of fills generated by the system under evaluation COR Number correct PAR Number partially correct (no partial credit was given in MUC-6 ) INC Number incorrec t SPU Number spuriou s MIS Number missin g NON Number non-committal (null fills generated by system that were also null in th e answer key) The rightmost columns in the report contain the values for the following extraction metrics : REC Recall = COR/POS PRE Precision = COR/AC T UND Undergeneration = MIS/POS OVG Overgeneration = SPU/ACT SUB Substitution = INC/(COR + INC ) ERR Error per response fill = (INC + SPU + MIS)/(COR + INC + SPU + MIS ) The first row at the bottom of the report is marked as the ALL OBJECTS row and gives the official overall tallies and the overall REC, PRE, UND, OVG, SUB, and ERR.
6:194	The tallies in the POS, ACT, COR, PAR, INC, SPU, MIS, and NON columns add up to the ALL OBJECTS values, but not all rows are considered . The rows that 293 are not considered are the object rows (i .e. , where the entry in the SLOT column is an object name) and the subtota l rows (since they simply sum up the tallies for the scored slots in each object) . The scores for the metrics in the ALL OBJECTS row are not computed by averaging the scores in the individual slot rows, but instead are computed on th e basis of the tally totals contained in the ALL OBJECTS row itself.
7:194	The remaining rows at the bottom of the report contain scores for the following metrics : TEXT-FILTER Text filtering (recall and precision for document detection ) F-MEASURES F-Measures (weighted combination of recall and precisio n scores in ALL OBJECTS row) The formulation of the text-filtering scores is based on the relevant/nonrelevant decisions made by th e system for the text as a whole, not on the basis of tallies in the slot rows . Some of the score category columns are use d to contain the tallies for these decisions across the test set : COR (decides relevant and relevant is correct), SPU (decides relevant and nonrelevant is correct), MIS (decides nonrelevant and relevant is correct), and NON (decide s nonrelevant and nonrelevant is correct) . Text filtering formulas are therefore as follows : REC = COR/(COR + MIS) ; PRE = COR/(COR + SPU); UND = MIS/(COR + MIS); OVG = SPU/(COR + SPU).
8:194	F-Measure scores corresponding to three different weightings of precision relative to recall are computed . The P&R value weights precision the same as recall ; the 2P&R value gives twice the weight to precision ; the P&2R value weights precision half as much as recall . The general formula is : (32 +1 .0)xPx R (03 2 x P)+ R The variation on the Scenario Template (ST) score report for Template Element (TE) is minor ; there is no template object and no text filtering score because the objects are generated independent of any domain (scenario) relevance criteria.
9:194	The variation for the Named Entity (NE) task is much more extensive because more diagnostic s are provided . The first table is the same as for ST and TE.
10:194	The second table provides scores for tas k subcategorization, e .g. there are rows for each of the type fills showing how well the system did on subcategorizing an entity as an organization as opposed to a person or location.
11:194	The third table reports the scores according to whic h part of the document the named entity appeared in . The sections considered were HL (headline), DD (documen t date), DATELINE, and TXT (text).
12:194	The Coreference (CO) task is quite different from the other three tasks and the output is also different . The score reports contain only recall and precision scores . The scoring method looks at linkages that define equivalenc e classes, and the important measure is how many linkages need to be added or taken away to get the correc t equivalence class . For a more detailed description of the algorithm used, please see the MITRE technical repor t entitled "A Model-Theoretic Coreference Scoring Scheme" by Marc Vilain, et al . It provides a solution to a difficul t evaluation problem and also offers an elegant definition of recall and precision . There are no F-Measures calculated because they have no sensible definition in this context . Please note that the scores for CO are ratios and produc e numbers less than 1 . To obtain the percentage, multiply by 100 . The scoring system output that is valuable fo r diagnostics on this task is the listing of equivalence classes in the key and response which also includes an indication of the key-to-response mappings (alignments).
